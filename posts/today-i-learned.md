# 오늘 배운 것

클로드 코드(Claude Code)와 함께 직접 블로그를 만들어 보았다. 프레임워크 없이 **HTML, CSS, JavaScript** 세 가지만으로 웹사이트가 어떻게 굴러가는지 몸으로 익힌 하루였다. 각자 맡은 역할이 명확히 나뉘어 있다는 점이 가장 인상 깊었다.

## HTML — 뼈대

HTML은 페이지의 **구조와 내용**을 담당한다. 제목, 문단, 목록, 링크처럼 "무엇이 있는가"를 정의하는 층이다.

```html
<article>
  <h1>글 제목</h1>
  <p>본문 내용입니다.</p>
</article>
```

`<article>`, `<header>`, `<nav>`, `<footer>` 같은 **시맨틱 태그**를 쓰면 사람이 봐도, 검색엔진이 봐도 의미가 분명해진다. 블로그에서는 글 목록과 본문 페이지의 뼈대를 HTML이 잡아 주었다.

## CSS — 옷

CSS는 그 뼈대에 **모양과 배치**를 입힌다. 색, 글꼴, 간격, 반응형 레이아웃이 모두 여기서 결정된다.

```css
:root {
  --color-bg: #ffffff;
  --color-text: #111111;
}
[data-theme="dark"] {
  --color-bg: #111111;
  --color-text: #dddddd;
}
```

특히 **CSS 변수(커스텀 프로퍼티)** 가 강력했다. 색을 변수로 한 번만 정의해 두면, `data-theme` 속성 하나만 바꿔도 사이트 전체가 다크 모드로 전환된다. 같은 HTML이라도 CSS만 바꾸면 완전히 다른 느낌이 된다는 걸 체감했다.

## JavaScript — 움직임

JavaScript는 페이지에 **동작과 상호작용**을 더한다. 정적인 문서를 살아 있는 앱처럼 만드는 층이다.

블로그에서 JS가 한 일을 정리하면:

| 기능 | 한 일 |
| --- | --- |
| 마크다운 파싱 | `.md` 텍스트를 HTML로 변환 |
| 포스트 목록 | `index.json`을 읽어 화면에 렌더링 |
| 검색·태그 필터 | 입력에 따라 목록을 실시간으로 거르기 |
| 다크 모드 토글 | 버튼 클릭 시 테마 전환 + 저장 |

```javascript
async function loadPost(slug) {
  const res = await fetch(`posts/${slug}.md`);
  return await res.text();
}
```

`fetch`로 파일을 불러오고, `async/await`로 그 결과를 기다린 뒤, DOM을 직접 바꿔 화면을 갱신한다. 사용자의 클릭과 입력에 반응하는 모든 부분이 JavaScript의 몫이었다.

## 정리

> HTML은 **뼈대**, CSS는 **옷**, JavaScript는 **움직임**.

세 가지가 각자의 역할에 충실하면서 서로 맞물릴 때 비로소 하나의 웹사이트가 완성된다는 걸 배웠다. 프레임워크는 결국 이 셋을 더 편하게 다루기 위한 도구일 뿐, 근본은 이 기초에 있다는 점을 잊지 말아야겠다.
